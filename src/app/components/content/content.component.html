Boundead

As a final project of the Computer Engineering degree, my friend Narcís Bustins Núñez and I made a 2.5D adventure/platform videogame in Unity. We worked for 6 months, spending all available hours on the project, and we are really proud and satisfied with the final result.

The game is composed by two levels, one that acted as an introduction and as a tutorial, and a final one with a final boss to conclude the game.

We modeled some of the objects we used, like the main player or the following buddy, but we mostly used assets from the Unity Store or other websites.

We programmed all the behaviors and systems of this game, that include:

  • Player

    • This system allows the control of the player in a 2.5D Space, with a horizontal and vertical movement.

    This includes all the main mechanics of movement: move, jump, crouch and dash.

  • Combat System

    • A system that would allow different entitites (player and enemies) to make, recieve damage and die.

    This system includes features to make damage with melee weapons, or throw damaging projectiles.

  • Enemies
    • Melee enemy: a humanoid that would follow the player and attack when is close to it.
    • Ranged enemy: a humanoid that would follow the player and throw at it when in range of attack.
    • Slime enemy: the fastest enemy that would follow the player and throw himself at it when in range.
    • Final boss: a humanoid with different attacks and abilities. It has different phases based on how much health it has, making it harder on low hp.
  • Experience System

    • A full system that allowed the player to earn experience points when killing enemies, and then a system that would allow the player to spend these points on unlockable abilities and upgrading player stats like max HP, armor, etc.

  • Other

    • And many others like Animations, Traps, Consumables, Maskarea System (the entity that follows the player), etc. Find more information in the documentation!

IsItRealEngine & RealDisunity Video Game Engine

As part of our Master's Degree program, we developed a small video game engine individually made from scratch in C++ and OpenGL. Mine was named IsItRealEngine and it could render 2D images and 3D models, as well as importing new models, allowed camera movement and management, configuration and monitoring panels.

IsItRealEngine Github repository

Once we developed our own video game engines, we grouped in pairs and created a more advanced engine. Ours was named RealDisunity Engine and it introduced frustum culling, scene management (save and load), game objects hierarchies, components management, materials creation, phong shader, panels docking, etc.

RealDisunity Github repository

Tesseract Video Game Engine

In the Master's degree, after the release of RealDisunity, we had to create a video game engine in collaboration with all the students of the masters, and we ended up creating an advanced engine called Tesseract Engine.

This engine would be the final engine that we would use for the creation of our own video game in the Master's degree.

Tesseract Engine Github repository

These are the features I worked on:

  • 2D Components
    • Component Transform 2D

      • Allowing a 2D GameObject to be rotated and placed at any position of the screen.

    • Component Image

      • Which allowed the visual representation of a 2D image or a color.

    • Component Text

      • A colorized multiline 2D text. Could be centered horizontally in 3 different positions: left, center or right.

  • Graphics
    • MSAA

      • Allowing a 2D GameObject to be rotated and placed at any position of the screen.

    • Unlit Shader

      • Which allowed the visual representation of a 2D image or a color.

    • Dissolve Shader

      • A colorized multiline 2D text. Could be centered horizontally in 3 different positions: left, center or right.

    • Chromatic aberration effect

      • A colorized multiline 2D text. Could be centered horizontally in 3 different positions: left, center or right.

  • Navigation Module

    • I created the Navigation Module, integrating Recast & Detour libraries to generate a Navigable Mesh from the Engine, and that would also allow a Navigation Agent to roam around the NavMesh, or Navigation Obstacles that would limit the NavMesh movement.

    • Component NavAgent

      • Allows a GameObject to roam a NavMesh. It had a maximum speed and acceleration, and an "Avoid obstacles" option could be set. This was used in the player controller and the enemies controller.

    • Component Obstacle

      • Allows a GameObject to modify the NavMesh in order to limit the movement through the NavMesh. The Obstacle can have different sizes and shapes: cylinder or a box shape.
