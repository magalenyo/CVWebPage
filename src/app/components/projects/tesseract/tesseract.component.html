Tesseract Video Game Engine

In the Master's degree, after the release of RealDisunity, we had to create a video game engine in collaboration with all the students of the masters, and we ended up creating an advanced engine called Tesseract Engine.

This engine would be the final engine that we would use for the creation of our own video game in the Master's degree.

Tesseract Engine Github repository

These are the features I worked on:

  • 2D Components
    • Component Transform 2D

      • Allowing a 2D GameObject to be rotated and placed at any position of the screen.

    • Component Image

      • Which allowed the visual representation of a 2D image or a color.

    • Component Text

      • A colorized multiline 2D text. Could be centered horizontally in 3 different positions: left, center or right.

  • Graphics
    • MSAA

      • Allowing a 2D GameObject to be rotated and placed at any position of the screen.

    • Unlit Shader

      • Which allowed the visual representation of a 2D image or a color.

    • Dissolve Shader

      • A colorized multiline 2D text. Could be centered horizontally in 3 different positions: left, center or right.

    • Chromatic aberration effect

      • A colorized multiline 2D text. Could be centered horizontally in 3 different positions: left, center or right.

  • Navigation Module

    • I created the Navigation Module, integrating Recast & Detour libraries to generate a Navigable Mesh from the Engine, and that would also allow a Navigation Agent to roam around the NavMesh, or Navigation Obstacles that would limit the NavMesh movement.

    • Component NavAgent

      • Allows a GameObject to roam a NavMesh. It had a maximum speed and acceleration, and an "Avoid obstacles" option could be set. This was used in the player controller and the enemies controller.

    • Component Obstacle

      • Allows a GameObject to modify the NavMesh in order to limit the movement through the NavMesh. The Obstacle can have different sizes and shapes: cylinder or a box shape.
